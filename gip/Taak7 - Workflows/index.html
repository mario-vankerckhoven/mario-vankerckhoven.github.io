Task 7: Workflows ~ Hugo vs WordPress

Thu, Oct 22, 2015

Wat moet je gebruiken ? Hugo of Wordpress ?

Voor een beginner is Wordpress een echte aanrader maar vanaf dat je reeds kennis hebt met html/css code raad ik het leren gebruiken van Hugo aan, aangezien hugo over meer aanpassingsmogelijkheden beschikt. Je moet echter wel een periode inlassen om de Directory die Hugo CreÃ«ert te begrijpen. Bij een WordPress site is dit niet het geval aangezien er duidelijk word verwoord wat er gebeurt wanneer je iets aanpast. Als je eenmaal de Hugo structuur door hebt, heeft WordPress gebruiken eigenlijk weinig zin.



Eigen ervaring

Hugo

Persoonlijk vind ik Hugo een redelijk lastige tool om te gebruiken aangezien er weinig tot geen documentatie te vinden is. De Documentatie die er wel is, is niet gebruiksvriendelijk vanaf je de nieuwe site hebt aangemaakt en lijkt gemaakt te zijn voor professionele gebruikers van Hugo. Echter vind ik wel dat Hugo een handige tool is in gebruik vanaf je de basishandelingen kunt gebruiken zoals Hugo Server, Hugo baseUrl, etc . Na de opdracht om met Hugo een website/blog te maken, vind ik dat je Hugo wel beter begrijpt qua mappenstructuur en verwijzingen.


WordPress

Persoonlijk vind ik WordPress een superhandige tool voor een beginner of iemand die nog nooit iets met webhosting/html&CSS heeft gedaan. Ook vind ik het wel handig dat je bij het kiezen van een startthema al direct een teaser te zien krijgt van hoe je website na 2 klikjes er al uit ziet. Alles op de WordPress site is makkelijk aan te passen aangezien het menu duidelijk verwoord wat elke aanpassing doet. Het publiceren van een WordPress blog is ook zeer makkelijk, er is letterlijk een knopje “Opslaan & publiceren”. Een ander voordeel daarentegen is ook dat de bezoekers van de site makkelijk een comment kunnen achterlaten.



Directory structuur Hugo

  • archetypes: Hier bevind zich het default.md bestand, lijkt onbelangrijk buiten het feit dat er in staat dat de taal GO word gebruikt
  • content: Alle eventuele post worden hierin aangemaakt, een subdirectory aanmaken is mogelijk om bepaalde posts te kunnen groeperen
  • layouts:
    • default: De html code in deze bestanden bestaat uit verwijzingen naar de html bestanden in de subdirectory ‘partials’. Ook word hier bepaalt hoe je posts standaard op het scherm verschijnen alvoor ze individueel geopent worden
    • partials: De basisopmaak-bestanden bevinden zich hier: header.html , head.html en footer.html. Eventuele aanpassingen die invloed hebben op al je headers/heads/footers doe je hier

  • static:

    • css: Hierin worden de kleuren, groottes en alle andere dingen in verband met opmaak gedefinieerd
    • fonts: Het lettertype dat een bepaald iets op je website gebruikt word vanaf hier geladen. Nagelang je font krijg je er ook leuke extra’s bij zoals icoontjes
    • images: Hierin worden alle eventuele images geplaatst die je op je website wilt gebruiken. tenzij je images laad via URL’s

  • config.toml: De basis van alles wat hugo doet staat hier (defaultwaarden worden wel verborgen).

  • public:Elke keer je een nieuwe website bouwt met hugo server –flag1 –flag2 , word hier de volledig gegenereerde website geplaatst
